# Shopbot — Telegram VPN shop-bot + админ-панель + мини-апп (Remnawave)

Чистый шаблон для развёртывания на новом проекте. Включает:
- **Telegram-бот** магазина VPN (продажа/продление подписок, триал, промокоды, рефералка, колесо фортуны, конкурсы).
- **Веб-админку** (Flask) — дашборд, настройки, ноды, рассылки, уведомления, поддержка.
- **Мини-апп / веб-кабинет** (FastAPI) — личный кабинет, покупка, мастер подключения.
- **Планировщик** — напоминания об истечении, автобэкапы, мониторинг, движок ре-энгейджмента.
- Интеграция с панелью **Remnawave**.

## Быстрый старт

```bash
cp .env.example .env
# сгенерировать и вписать SHOPBOT_SECRET_KEY:
python3 -c "import secrets;print(secrets.token_hex(32))"

docker compose up -d --build
```

БД (`users.db`) создаётся **автоматически при первом старте** (пустая, схема ставится сама). Пользователи добавляются через бота.

## Что настроить после запуска (в АДМИН-ПАНЕЛИ, не в коде)

- Токен Telegram-бота, username, admin id.
- Адрес и токен панели **Remnawave**, хосты/ноды.
- Платёжные системы (YooKassa / CryptoBot / Heleket / Platega / TON / Stars).
- Домен мини-аппа (`webapp_domen`), логотип, тексты, тарифы.
- Рефералка, промокоды, уведомления, колесо, конкурсы.

## ⚙️ Брендинг (заменить под свой проект)

Шаблон снят с рабочего проекта; часть брендинга — текстом в коде/ассетах, замени под себя:
- Название «Alma VPN» в некоторых местах (напр. `src/shop_bot/webapp/manifest.webmanifest`, дефолты настроек) → своё.
- Ассеты: логотип, welcome-видео, картинки, звуки колеса (`src/shop_bot/webapp/module/wheel/*.mp3`) — замени на свои.
- Функциональные ссылки на чужую инфраструктуру уже вычищены (кнопка «Сайт» берётся из настройки домена).

## Структура

```
src/shop_bot/
  bot/            — Telegram-бот (aiogram)
  webapp/         — мини-апп/кабинет (FastAPI) + app.html
  webhook_server/ — админка (Flask) + вебхуки платежей + шаблоны
  data_manager/   — БД (SQLite), схема, планировщик, уведомления, ре-энгейджмент
  modules/        — Remnawave API и пр.
  support_bot/    — бот поддержки
```

## Безопасность

- `.env`, `*.db`, `backups/`, любые данные пользователей — в `.gitignore`, в репозиторий не попадают.
- Настройки и секреты (токены/ключи) хранятся в БД (через админку), не в коде.
