# adminka_work — деплой и передача задачи

Чистый шаблон shopbot (Telegram VPN-магазин + админка + мини-апп, интеграция с Remnawave).
Снят с рабочего проекта и обезличен. Этот документ — для продолжения задачи в новом чате.

## 🎯 ЦЕЛЬ ЗАДАЧИ

На **существующем сервере** уже стоит и работает **Remnawave** (VPN-панель), биллинг сейчас — **SHM**.
Надо: **выкинуть SHM**, развернуть эту админку (`adminka_work`) как новый биллинг/магазин и **связать её с существующей Remnawave**. Домен и инфраструктура уже есть.

## ✅ Что в репозитории (чисто)

- Весь код: `src/shop_bot/` — bot (aiogram) / webapp (FastAPI, мини-апп) / webhook_server (Flask, админка + вебхуки платежей) / data_manager (SQLite, схема, планировщик, уведомления, движок ре-энгейджмента) / modules (Remnawave API) / support_bot.
- `Dockerfile`, `docker-compose.yml`, `install.sh` (полный установщик), `.env.example`, `pyproject.toml` (зависимости).
- Звуки колеса фортуны (`src/shop_bot/webapp/module/wheel/*.mp3`) — куплены, оставлены.
- **НЕТ:** данных пользователей, секретов, `.env`, Alma-инфры/доков/скриптов. БД создаётся ПУСТОЙ при первом старте (схема ставится сама).

## 🚀 Развёртывание

**Почти одна команда** (`install.sh` ставит Docker + nginx + certbot, клонирует репо, создаёт docker-сеть `remnawave-network`, выпускает SSL, поднимает контейнер, генерит `SHOPBOT_SECRET_KEY`):

```bash
git clone git@github.com:ponoroshca/adminka_work.git && cd adminka_work && ./install.sh
```

### ⚠️ Нюансы (обязательно учесть)
1. **Репо ПРИВАТНЫЙ** → на сервере нужен доступ к GitHub для clone (SSH deploy-key или PAT-токен). Иначе — залить файлы напрямую (scp/rsync).
2. **Remnawave уже есть на сервере** → docker-сеть `remnawave-network` скорее всего уже создана панелью (compose к ней цепляется как external). Если shopbot на ТОМ ЖЕ сервере — свяжется по внутренней сети; если панель отдельно — по её публичному API.
3. **Домен + SSL** — install.sh спросит домен/email и выпустит сертификат. Если nginx/домен уже настроены под SHM — аккуратно, не сломать существующее (отдельный server-block/поддомен под shopbot).
4. **Порты** shopbot: `127.0.0.1:1488` (мини-апп) + `127.0.0.1:8000` (админка) — наружу через nginx 443.

### После установки (в АДМИН-ПАНЕЛИ)
- Вход: дефолт `admin` / `admin` → **СРАЗУ сменить** (см. аудит безопасности Alma — пароль панели plaintext, поставить свой; в идеале потом хеш).
- Вписать: токен Telegram-бота + username + admin id; **адрес и токен API Remnawave** + хост(ы)/сквад; платёжки (YooKassa/CryptoBot/Heleket/Platega/TON/Stars); домен мини-аппа (`webapp_domen`); тарифы; тексты.

## Полезные факты из исходного проекта (Alma) — применимо к архитектуре
- shopbot = ВИТРИНА/биллинг; сам VPN — это Remnawave (панель) + ноды. shopbot управляет ключами через API панели.
- Запуск контейнера: `python -m shop_bot` (бот+админка+веб-апп в одном процессе-контейнере).
- Деплой правок: `git pull`/`checkout` → `docker compose up -d --build` (для .py/схемы), app.html читается per-request (без rebuild).
- Уведомления — реестр (`data_manager/notifications.py`) + вкладка «Уведомления»; движок ре-энгейджмента — `reengagement.py` (мастер-тумблер `reengage_enabled`).
- Рефералка двусторонняя (реферер % + приглашённый скидка `referral_discount`).
